1️⃣ Javaの基本・データ型・演算子

🎯 試験で問われるポイント

  • プリミティブ型のサイズと範囲（byte: -128〜127, int: 約±21億）
  • 暗黙的型変換（拡大変換）とキャスト（縮小変換）
  • 前置/後置インクリメント（++a vs a++）の違い
  • 整数除算の結果（int/int=int）
  • short・byteへの代入はキャストが必要（計算結果はint以上）

⚠️ 初学者が間違えやすい点

float代入にはFサフィックスが必要 float f = 3.14; // コンパイルエラー！3.14はdouble型
float f = 3.14F; // これが正解
Java: データ型の基本
byte b = 100;           // -128〜127
int i = 2_000_000;    // アンダースコア区切りOK（Java7以降）
long l = 9999999999L; // Lサフィックス必須
double d = 3.14;      // デフォルトはdouble
float f = 3.14F;      // Fサフィックス必須
char c = 'A';          // シングルクォート
boolean ok = true;    // true / false のみ

// 拡大変換（自動）
int x = b; // byteはintに自動変換OK

// 縮小変換（キャスト必須）
byte b2 = (byte) i; // 明示的キャストが必要

2️⃣ 条件分岐・繰り返し処理

🎯 試験で問われるポイント

  • switchのfall-through（breakなしで次のcaseに流れる）
  • ラベル付きbreak/continue（外側ループを操作）
  • for-eachループの使い方と制限（インデックス操作不可）
  • while vs do-while（do-whileは必ず1回実行）
Java: switch文のfall-through
int x = 2;
switch (x) {
    case 1:
        System.out.println("one");
        break;
    case 2:           // breakなし→fall-through
    case 3:
        System.out.println("two or three");
        break;
    default:
        System.out.println("other");
}
// 出力: two or three
⚠️ switchで使える型 byte, short, char, int, String, enum（Java7以降でString対応）。long, floatは使えません。

3️⃣ 配列

🎯 試験で問われるポイント

  • 配列の初期値（int→0, boolean→false, 参照型→null）
  • length はフィールド（括弧なし）
  • 配列の代入は参照コピー（同じオブジェクトを指す）
  • Arrays.sort()・Arrays.toString()・Arrays.copyOf()の使い方
  • ArrayIndexOutOfBoundsException の発生条件
Java: 配列の基本操作
// 宣言と初期化
int[] arr1 = new int[5];          // 全要素0で初期化
int[] arr2 = {1, 2, 3, 4, 5};  // 初期値あり

// lengthはフィールド（括弧なし！）
System.out.println(arr2.length);  // 5

// Arrays.sortとtoString
int[] nums = {3, 1, 4, 1, 5};
java.util.Arrays.sort(nums);
System.out.println(java.util.Arrays.toString(nums));
// [1, 1, 3, 4, 5]

// 参照コピーに注意！
int[] a = {1, 2, 3};
int[] b = a;  // 同じ配列を指す
b[0] = 99;
System.out.println(a[0]);  // 99（aも変わる！）

4️⃣ String / StringBuilder

🎯 試験で問われるポイント

  • ==はアドレス比較、equalsは値比較
  • Stringはイミュータブル（変更不可）
  • substring(begin) / substring(begin, end)の動作
  • StringBuilderのappend・insert・delete・reverse
  • String.valueOf()・toString()の変換
Java: String主要メソッド
String s = "Hello World";

s.length()          // 11
s.charAt(0)         // 'H'
s.indexOf("o")      // 4（最初に見つかった位置）
s.lastIndexOf("o") // 7
s.substring(6)      // "World"
s.substring(0, 5)  // "Hello"
s.toUpperCase()     // "HELLO WORLD"
s.toLowerCase()     // "hello world"
s.trim()            // 前後の空白削除
s.replace("l","L") // "HeLLo WorLd"
s.contains("World")// true
s.startsWith("He") // true
s.endsWith("ld")  // true
s.isEmpty()         // false
s.split(" ")        // ["Hello", "World"]

5️⃣ クラス・コンストラクタ・static

🎯 試験で問われるポイント

  • staticフィールドはクラス全体で共有（インスタンスは各自の値を持つ）
  • staticメソッドからインスタンスメンバは参照不可
  • コンストラクタはvoid不要・クラス名と同名
  • this()・super()はコンストラクタ第1行目のみ
  • デフォルトコンストラクタは引数ありを定義すると追加されない
Java: クラスとstatic
class BankAccount {
    static int totalAccounts = 0;  // クラス変数（共有）
    String owner;                   // インスタンス変数（各自）
    double balance;

    BankAccount(String owner, double balance) {
        this.owner = owner;
        this.balance = balance;
        totalAccounts++;
    }

    static int getTotalAccounts() {
        return totalAccounts; // staticメンバのみOK
    }
}

BankAccount a1 = new BankAccount("Alice", 1000);
BankAccount a2 = new BankAccount("Bob", 2000);
System.out.println(BankAccount.getTotalAccounts()); // 2

6️⃣ 継承・オーバーライド・ポリモーフィズム

🎯 試験で問われるポイント

  • インスタンスメソッドはオーバーライドされる（実行時型）
  • フィールドとstaticメソッドは参照型で解決（コンパイル時型）
  • オーバーライドのルール（アクセス修飾子は広げるのみ可）
  • super()はコンストラクタ呼び出し、コンストラクタ実行順は親→子
  • finalクラス・finalメソッドは継承/オーバーライド不可
Java: ポリモーフィズムの例
class Animal {
    String name = "Animal";
    void speak() { System.out.println("..."); }
}
class Dog extends Animal {
    String name = "Dog";         // フィールドはhiding
    @Override
    void speak() { System.out.println("Woof"); } // メソッドはoverride
}

Animal a = new Dog();
a.speak();           // "Woof"（実行時型Dogのメソッド）
System.out.println(a.name);  // "Animal"（参照型Animalのフィールド）

7️⃣ インターフェース・抽象クラス

🎯 試験で問われるポイント

  • インターフェースは多重実装可能（abstractクラスは1つだけ継承）
  • インターフェースにはdefaultメソッド（Java8）・staticメソッドが書ける
  • 抽象クラスはインスタンス化不可、コンストラクタを持てる
  • インターフェースのメソッドはpublic abstract（省略可）
  • インターフェースのフィールドはpublic static final（省略可）

8️⃣ 例外処理

🎯 試験で問われるポイント

  • チェック例外 vs 非チェック例外（RuntimeExceptionの子孫かどうか）
  • try-catch-finally の実行順序
  • finallyはreturnよりも先に実行される
  • 多重catch（catch (A | B e)はJava7以降）
  • try-with-resources（AutoCloseable実装クラスを自動close）
Java: 例外処理の例
// try-with-resources（Java7以降）
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    e.printStackTrace();
}
// tryブロック終了時にbrが自動的にcloseされる

// マルチキャッチ（Java7以降）
try {
    // 処理
} catch (IOException | SQLException e) {
    System.out.println("エラー: " + e.getMessage());
}

🔟 ラムダ式・Stream API

🎯 試験で問われるポイント

  • 関数型インターフェースとは（抽象メソッドが1つのinterface）
  • Predicate / Function / Consumer / Supplier の違い
  • Stream APIの中間操作と終端操作
  • Streamは再利用不可（一方向パイプライン）
  • メソッド参照（ClassName::methodName）
Java: Stream APIの例
List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "Anna");

// Aで始まる名前を大文字で収集
List<String> result = names.stream()
    .filter(n -> n.startsWith("A"))   // 中間操作
    .map(String::toUpperCase)           // 中間操作（メソッド参照）
    .sorted()                           // 中間操作
    .collect(Collectors.toList());      // 終端操作

System.out.println(result); // [ALICE, ANNA]

// 集計
long count = names.stream().filter(n -> n.length() > 3).count();
System.out.println(count); // 2（Alice, Charlie）

1️⃣1️⃣ モジュール基礎

🎯 試験で問われるポイント

  • Java 9で導入されたモジュールシステム（JPMS）
  • module-info.java ファイルの基本構文
  • requires：別モジュールへの依存宣言
  • exports：外部に公開するパッケージの宣言
  • Silver試験では基本的な構文理解が求められる（深い実装は不要）
module-info.java
// モジュール宣言ファイル（module-info.java）
module com.example.myapp {
    requires java.sql;        // 依存するモジュール
    requires java.logging;
    exports com.example.api;  // 外部に公開するパッケージ
}
